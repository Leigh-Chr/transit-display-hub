# Internationalisation (i18n)

Transit Display Hub uses [Transloco](https://jsverse.gitbook.io/transloco)
for runtime FR / EN switching. Strings live in
`frontend/src/assets/i18n/<lang>.json`.

Supported languages today:

- `fr.json` — French (default, tutoiement / informal "tu" form)
- `en.json` — English

The runtime switch is exposed via the language menu in the admin
layout and via the `LocaleService`. The default language is French
(see `frontend/src/app/core/i18n/transloco.providers.ts`).

## Adding a key

1. Add the new key to **both** `fr.json` and `en.json` (same key
   path, same nested structure).
2. Reference the key in templates via the `transloco` pipe:
   ```html
   {{ 'feature.section.label' | transloco }}
   ```
3. For `aria-label` or any other attribute binding:
   ```html
   <button [attr.aria-label]="'feature.section.aria' | transloco">…</button>
   ```
4. Run tests: `cd frontend && npm test`. Component specs that mount
   templates referencing Transloco keys must import
   `testTranslocoModule` from `frontend/src/test-translations.ts`.
5. Run `npx knip` from `frontend/` to confirm no unused symbol.

## Adding a language

1. Create `frontend/src/assets/i18n/<lang>.json` mirroring `en.json`
   exactly (same key paths, translated values).
2. Register the language in
   `frontend/src/app/core/i18n/transloco.providers.ts`:
   ```typescript
   export const TRANSLOCO_AVAILABLE_LANGS = ['fr', 'en', '<lang>'] as const;
   ```
3. The language selector picks it up automatically.

## Translation conventions

- **French**: tutoiement (use "tu", not "vous"). Friendly, direct.
- **All languages**: avoid technical jargon in user-facing strings.
  "GTFS feed" is fine in the admin UI but not in kiosk strings.
- **Pluralization**: use the Transloco
  [`pipe(plural)`](https://jsverse.gitbook.io/transloco/plugins/pluralization)
  plugin when needed.
- **Interpolation**: prefer `{{ value }}` placeholders over string
  concatenation in TS code (`transloco.translate(key, { value })`).

## Removing a key

1. Confirm no usage: `grep -rn "myFeature.dead.key" frontend/src`.
   Also check template literals (`` `prefix.${dynamic}` ``) before
   declaring a key dead.
2. Remove from `fr.json` and `en.json` (and any other locale).
3. Re-run `npx knip` to confirm no orphan.

## Capturing screenshots (multi-language)

```bash
cd frontend
# English screenshots
LANG=en SCREENSHOTS_ENABLED=1 npx playwright test screenshots.spec.ts

# French
LANG=fr SCREENSHOTS_ENABLED=1 npx playwright test screenshots.spec.ts
```

Output lands in `docs/screenshots/`. Commit the diff like any other
change. See [`docs/screenshots/README.md`](screenshots/README.md)
for the full regeneration workflow.

## Reporting translation issues

Open an issue with the `i18n` label. Include:

- Source key (e.g., `kiosk.banner.delay`)
- Current translation
- Suggested fix
- Locale (`fr`, `en`, …)
