## House rules

- **No "first stable release" on every commit.** Announce the
  1.0.0 cut once. Subsequent posts mention what changed since
  1.0.0, not the project's existence.
- **Lead with the differentiator.** All-in-one GTFS back-office +
  WCAG 2.2 AA + Pi installer is the unique combination — the
  spec coverage alone isn't enough to stand out.
- **Always link to the GitHub repo, never to a marketing
  domain.** The user explicitly declined a separate landing
  page; the README is the canonical entry point.
- **Translate when the audience expects it.** French for
  `transport.data.gouv.fr` and Devoxx FR; English everywhere
  else.
